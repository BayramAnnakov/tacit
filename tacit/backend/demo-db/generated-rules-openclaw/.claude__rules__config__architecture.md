description: Architecture conventions
---

- When validating Discord `activityType` in Zod schemas, use an explicit union of allowed literal values (e.g., `z.union([z.literal(0), z.literal(1), z.literal(2), z.literal(3), z.literal(5)])`) instead of `z.number().min(0).max(5)`, because Discord activity type 4 is unused/invalid and a contiguous range check would allow it.
